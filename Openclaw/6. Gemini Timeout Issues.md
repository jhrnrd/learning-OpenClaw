# Issue: Gemini API Timeout on OpenClaw (VPS)

## Status
⚠️ Unresolved — Workaround in progress

## Date
2026-05-02

## Summary
OpenClaw successfully recognizes `gemini/gemini-2.0-flash` as the agent model,
but times out when making actual API calls to Google's OpenAI-compatible endpoint.

## Symptoms
- Logs show: `agent model: gemini/gemini-2.0-flash` ✅
- Logs show: `fetch timeout reached; aborting operation` ❌
- Telegram bot shows: `⚠️ Something went wrong while processing your request`
- Free tier also hits: `All models are temporarily rate-limited`

## Root Cause (Suspected)
Google's OpenAI-compatible Gemini endpoint may have geographic restrictions
or inconsistent routing when accessed from Tencent JKT cloud infrastructure.

The endpoint in question:
```
https://generativelanguage.googleapis.com/v1beta/openai
```

This endpoint is not Google's native Gemini API — it's a compatibility layer
that mimics OpenAI's API format. This layer appears to be less stable than
the native Gemini API, especially from certain cloud regions.

## What Was Tried

### Attempt 1: Gemini 1.5 Flash
```json
"id": "gemini-1.5-flash"
```
**Result:** Failed — model retired in 2026, returns 404.

### Attempt 2: Gemini 2.0 Flash
```json
"id": "gemini-2.0-flash"
```
**Result:** Model recognized but API calls timeout from VPS.

### Attempt 3: Clearing plugin cache
```bash
docker exec -it --user node openclaw rm -rf /home/node/.openclaw/plugin-runtime-deps
docker restart openclaw
```
**Result:** Fixed unrelated json5 package error, did not fix timeout.
