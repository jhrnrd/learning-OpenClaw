## What Was Done

### 1. VPS Provisioned
- Purchased on SumoPod
- Provider: Tencent, Region: JKT
- OS: Ubuntu 22.04 LTS

### 2. Initial Server Setup
```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Reboot to load new kernel
sudo reboot
```

### 3. Firewall Setup
```bash
sudo ufw allow OpenSSH
sudo ufw enable
sudo ufw status
```

### 4. Docker Installed
```bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker ubuntu
```

### 5. Docker Compose Installed
```bash
sudo apt install docker-compose-plugin -y
```
**Version:** Docker 29.4.2, Compose v5.1.3

### 6. OpenClaw Deployed
```bash
mkdir -p ~/.openclaw
docker run -d \
  --name openclaw \
  --restart unless-stopped \
  -p 18789:18789 \
  -v ~/.openclaw:/home/node/.openclaw \
  ghcr.io/openclaw/openclaw:latest
```
Note: `--restart unless-stopped` ensures OpenClaw restarts automatically on server reboot.

### 7. OpenClaw Configured
```bash
nano ~/.openclaw/openclaw.json
```
- Set gateway token
- Set Telegram bot token
- Set Gemini API key
- Pointed model to `gemini/gemini-2.0-flash`

### 8. Telegram Connected
```bash
# Add channel
docker exec -it --user node openclaw openclaw channels add \
  --channel telegram --token YOUR_BOT_TOKEN

# List pairing requests (after sending /start to bot)
docker exec -it --user node openclaw openclaw pairing list telegram

# Approve yourself
docker exec -it --user node openclaw openclaw pairing approve telegram YOUR_CODE
```

### 9. New Telegram Bot Created
Created a separate bot for VPS via @BotFather to avoid conflict with local laptop instance. Old bot remains for local dev/testing.

## Issues Encountered

### Issue 1: Wrong SSH username
- **Problem:** Tried `ssh root@IP`, got permission denied
- **Fix:** Use `ssh ubuntu@IP` instead — SumoPod default user is `ubuntu`

### Issue 2: json5 package missing
- **Problem:** Telegram channel kept crashing with `Cannot find package 'json5'`
- **Fix:** Clear plugin cache and restart
```bash
docker exec -it --user node openclaw rm -rf /home/node/.openclaw/plugin-runtime-deps
docker restart openclaw
```

### Issue 3: Telegram bot conflict
- **Problem:** Bot token was still connected to local laptop's OpenClaw
- **Fix:** Created new bot via @BotFather for VPS, stopped local OpenClaw

### Issue 4: Config overwritten by OpenClaw
- **Problem:** After adding Telegram channel, OpenClaw rewrote the config and dropped `models` and `agents` sections
- **Fix:** Always recheck and rewrite full config after any `openclaw channels add` command

### Issue 5: Gemini API timeout
- **Problem:** Gemini 2.0 Flash recognized but API calls timeout from VPS
- **Status:** Unresolved — see `docs/gemini-timeout-issue.md`
- **Next step:** Enable Gemini billing or switch to Kimi

### Issue 6: Groq rate limits
- **Problem:** Free tier rate limits hit immediately on always-on use
- **Decision:** Not suitable for production agent use on free tier
